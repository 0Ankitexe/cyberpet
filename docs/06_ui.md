# 06 — User Interface (TUI)

> How the terminal UI works — the pet, brain screen, and scan screen.

---

## Overview

CyberPet's UI is a **Textual** (Python TUI framework) application. It connects to the running daemon via a Unix socket event stream and renders live data.

```bash
cyberpet pet   # Launch the TUI
```

The TUI does **not** run inside the daemon — it's a separate process that subscribes to `/var/run/cyberpet_events.sock` and receives every event as a JSON line.

### Pet Moods

The pet's ASCII face changes based on system state:

| Mood | Face | Triggered When |
|------|------|---------------|
| `happy` | `(◕‿◕)` | Normal system, no threats |
| `curious` | `(◉_◉)` | Slight anomaly, watching |
| `alert` | `(◣_◢)` | Suspicious activity |
| `angry` | `(ò_ó)` | Active threat detected |
| `sad` | `(◞‸◟)` | False positive made, feeling guilty |
| `scared` | `(⊙_⊙)` | High anomaly score or lockdown |
| `sleepy` | `(-‸-)` | Very low activity, quiet system |

The mood is calculated from:
- Current threat level (from threat history in state vector)
- Recent RL decisions and outcomes
- RL model state (warmup / learning-safe / full)

### Key Bindings

| Key | Action |
|-----|--------|
| `Q` / `Ctrl+C` | Quit |
| `B` | Open Brain Screen (RL detail view) |
| `S` | Open Scan Menu |
| `L` | Tail log in terminal |
| `?` | Help |
| `ESC` | Go back / close modal |

### Live Updates

The main TUI refreshes from several sources:

| Data | How Often | Source |
|------|-----------|--------|
| System stats (CPU, RAM) | every 2s | Polls `PetState` |
| Pet mood | every 2s | Recalculated from state |
| Speech bubble | every 10s | Random or event-triggered |
| Scan progress | every 0.3s | Polls active scan event queue |
| Events (threats, RL) | immediate | Event stream socket |

### What Each Section Shows

**Reward Trend:** ASCII sparkline of the last 50 reward values. `█` = positive reward, `▒` = negative. Helps you see if the model is improving.

**Action Distribution:** Bar chart of how often each of the 8 actions has been chosen. A healthy model early on should show mostly ALLOW (0) and LOG_WARN (1). More variety appears as training progresses.

**Brain Status:** Live RL engine state — steps, IQ, training phase, FP rate. FP rate > 10% means the model is being too aggressive.

**Decision Log:** Scrollable list of recent RL decisions with timestamps, action taken, and the explanation generated by `rl_explainer.py`.

### Threat Actions (in scan screen)

When a threat is found, three buttons appear inline:

| Button | What It Does |
|--------|-------------|
| **🔒 Quarantine** | Moves file to vault, records in scan history, FP memory |
| **✅ Mark Safe** | Adds sha256 to FP memory, will never be flagged again |
| **👁 View Details** | Shows full threat record: score, category, matched YARA rules, entropy |

### Background Scanning

Pressing `ESC` during a scan **does not stop it**. The scan continues in the background. The main TUI shows live progress in the scan section (bottom right). Press `S` again to jump back to the scan screen.

## Threat Action Modal (`ui/threat_action.py`)

When a `THREAT_DETECTED` event arrives via the event stream (e.g., during a background scan), a modal pops up:

```
┌──── ⚠ Threat Detected ──────────────────────────────────┐
│                                                          │
│  File:     /tmp/suspicious.sh                           │
│  Score:    82 / 100                                     │
│  Category: dropper                                      │
│  Reason:   High entropy + YARA rule "reverse_shell"     │
│                                                          │
│  [🔒 Quarantine]   [✅ Mark Safe]   [⏭ Dismiss]         │
└──────────────────────────────────────────────────────────┘
```

This modal is non-blocking — other events continue arriving and are queued.

## Event Log (main TUI)

The bottom-left shows a scrolling log of recent events:

```
[03:14:22] ⚠  suspicious cmd: rm -rf /tmp/data
[03:12:44] 🤖 RL: ALLOW — system stable (+1.0)
[03:10:12] 🔒 Quarantined: /tmp/miner.sh (score 91)
[03:08:55] 📡 New external connection: 192.168.1.1:443
[03:05:11] ✅ Scan complete: 2,394 files, 0 threats
```

Events are colour-coded by severity. `ALLOW` events are hidden by default (too noisy) but can be enabled.
